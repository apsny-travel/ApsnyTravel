# Review Prompt Template

Use this for code/content reviews to keep the site within guardrails.

1) Scope  
- State what is being reviewed (files, routes, capsules).

2) Findings (ordered by severity)  
- Bugs/functional risks.  
- Spec/guardrail violations (stack, SSG, capsule-first).  
- Missing data or invalid deep-links/graph targets.  
- UX/clarity issues vs UI guidelines.

3) Questions/assumptions  
- Note uncertainties or required confirmations.

4) Change summary (brief)  
- What changed and where.

5) Tests/checks  
- What was run (build, lint, manual checks).  
- Remaining gaps or untested areas.
